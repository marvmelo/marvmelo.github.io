After some thought, my pair and I decided that we wanted a game where you could pilot a ship in 2D and shoot at enemy ships while they shoot back at you.You start the game with 5 lifes, which is the maximum you can have. Every time you get shot, you lose a life; every time you shoot an enemy, you gain 50 points. We also have some power-ups. There is a life power-up, that restore a life up to 5, and a shield power-up, that protects you against one single bullet. We thought about adding a speed power-up, but that would be more annoying than helpful.

After we split the task, I got to do mechanics of the game. I was mainly OOP code. There are 6 classes that make the game work. They are: Spacecraft(), Player(Spacecraft), Enemy(Spacecraft), Projectile(), PowerUp(), Sprites().

The most important class is Sprites, it is responsible for managing all other classes. Here is the data that is inside of it:

        
class Sprites():


    def __init__(self, player, score):

        self.enemy_projectiles = []
        self.powerups = []
        self.player_projectiles = []
        self.enemies = []
        self.player = player
        self.score = score

As you can see, it contains some lists that contains objects of the other classes. The list has elements appended and removed as they appear and disappear on the screen. The Sprites() class has 2 methods that are called every frame: update(...) and draw(...). They iterates over every list of objects and, respectively, calls the method with the same name that every class has implemented.
